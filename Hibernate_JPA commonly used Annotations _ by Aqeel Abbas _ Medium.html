Java Persistence API (JPA) é um padrão Java que fornece um conjunto de interfaces e anotações para acessar e gerenciar bancos de dados relacionais em aplicativos Java. JPA é parte do Java EE (Enterprise Edition) e também está disponível como uma tecnologia autônoma para Java SE (Standard Edition).

O JPA fornece uma API simples e fácil de usar para acessar bancos de dados relacionais sem escrever instruções SQL. Ele permite que os desenvolvedores mapeiem objetos Java para tabelas de banco de dados e executem operações CRUD (Create, Read, Update, Delete) neles. O JPA também oferece suporte a uma ampla gama de fornecedores de banco de dados e fornece uma maneira independente de fornecedor para acessar bancos de dados.

Neste artigo, apresentaremos algumas das anotações JPA mais comumente usadas com exemplos de código.

Anotação @Id

A @Idanotação é usada em Java para marcar um campo como a chave primária de uma entidade. Ela é tipicamente usada em conjunto com um framework ORM (Object-Relational Mapping) como o Hibernate, que mapeia objetos Java para tabelas de banco de dados relacionais.

@Anotação gerada

A @GeneratedValueanotação é usada em Java para gerar automaticamente valores para colunas de chave primária em uma tabela de banco de dados. Essa anotação é normalmente usada em conjunto com a @Idanotação para indicar que um campo específico é a chave primária para uma tabela.

  1. strategy: Este parâmetro especifica a estratégia a ser usada para gerar a chave primária. Há quatro estratégias disponíveis: AUTO, IDENTITY, SEQUENCE, e TABLE.

@Anotação de tabela

A @Tableanotação é usada em Java para especificar os detalhes da tabela do banco de dados para a qual uma entidade (classe) específica deve ser mapeada. Esta anotação pode ser usada para especificar o nome da tabela, seu esquema, catálogo, índices e outros detalhes específicos do banco de dados.

A @Tableanotação pode ter os seguintes parâmetros:

  1. name: Este parâmetro especifica o nome da tabela do banco de dados para a qual a entidade deve ser mapeada. Se não for especificado, o nome padrão será o nome da classe da entidade.
  2. schema: Este parâmetro especifica o nome do esquema (banco de dados) ao qual a tabela pertence.
  3. catalog: Este parâmetro especifica o nome do catálogo (banco de dados) ao qual a tabela pertence.
  4. uniqueConstraints: Este parâmetro é usado para especificar uma ou mais restrições exclusivas na tabela.
  5. indexes: Este parâmetro é usado para especificar um ou mais índices na tabela.

Aqui está um exemplo de como usar a @Tableanotação com o nameparâmetro:

@Table(name = "users") 
public  class  User { 
    @Id 
    @GeneratedValue(strategy = GenerationType.IDENTITY) 
    private Long id; 

    private String username; 
    private String password;

 }

@Column Annotation

The @Column annotation is used in Java to specify the details of a particular column in a database table that a field should be mapped to. This annotation can be used to specify the name of the column, its data type, length, precision, scale, and other database-specific details.

The @Column annotation can take the following parameters:

  1. name: This parameter specifies the name of the database column that the field should be mapped to. If not specified, the default name is the name of the field.
  2. nullable: This parameter specifies whether the column can contain null values or not.
  3. unique: This parameter specifies whether the column must contain unique values or not.
  4. length: This parameter specifies the maximum length of the column (for string columns).
  5. precision: This parameter specifies the precision of the column (for numeric columns).
  6. scale: This parameter specifies the scale of the column (for numeric columns).
  7. insertable: This parameter specifies whether the column should be included in SQL insert statements or not.
  8. updatable: This parameter specifies whether the column should be included in SQL update statements or not.

    @Column(name = "username", nullable = false)
    private String username;

    @Column(name = "password", nullable = false)
    private String password;
    // other fields and methods omitted for brevity
}

In this example, the User class represents a table in a database. The @Entity annotation is applied to the class to indicate that it should be mapped to a database table. The @Table annotation is also applied to the class with the name parameter set to "users". The username and password fields are marked with the @Column annotation with the name parameter set to "username" and "password", respectively. The nullable parameter is set to false for both columns, which means that they cannot contain null values. This means that the username and password fields will be mapped to non-null columns in the "users" table in the database.

If this parameter is set to true, the foreign key column will be included in UPDATE statements. If it is set to false, the foreign key column will not be included in UPDATE statements.

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne
    @JoinColumn(name = "department_id", nullable = false)
    private Department department;
    // other fields and methods omitted for brevity
}
@Entity
@Table(name = "departments")
public class Department {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    @OneToMany(mappedBy = "department")
    private List<Employee> employees;
    // other fields and methods omitted for brevity
}

The @JoinColumn annotation is also applied to the department field with the name parameter set to "department_id", which specifies the name of the foreign key column in the employees table. The nullable parameter is set to false, which specifies that the foreign key column should not allow null values.

The Department class also has a one-to-many relationship with the Employee class, which is represented by the employees field. The @OneToMany annotation is applied to the field with the mappedBy parameter set to "department", which specifies that the relationship is mapped by the department field in the Employee class.
